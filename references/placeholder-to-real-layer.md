# 占位符→真实层转换技术

> 本文件由十轮自检 🅡130 创建（原文件索引存在但文件缺失）
> 创建时间: 2026-06-21

## 背景

在 Phase 14（7层结构对齐）中，技能架构经历了从"3层"到"7层"的扩展。
"占位符→真实层"指的是将原本只是表格中的一行（如"灵魂层: 已有OpenClaw"）
转化为具有实际内容、加载路径、运维规则的真实层级。

## 可复用模式

### 模式1：占位符→文件
1. 在表格中新加一行（如"创意层 CREATIVE.md"）
2. 创建对应文件（`~/.hermes/memories/CREATIVE.md`）
3. 定义 TTL 和生命周期规则
4. 集成到检索/预加载管道

### 模式2：继承→增强
1. 识别已有但不完整的组件（如 OpenClaw SOUL.md）
2. 评估其现有内容和可用性
3. 扩写/替换为完全版（如行为价值观+GT优先级）
4. 保留兼容性（profile SOUL.md 仍可用）
